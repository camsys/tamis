Introducing TAMIS

Welcome to ADOT&PF's Transportation Asset Management Information System Proof of Concept (TAMIS POC). This tool is designed to illustrate the potential benefits of bringing together data from disparate ADOT&PF data sources to support integrated, spatially referenced queries with the aim of providing more holistic information for improved asset management and other strategic decisions.

Signing In

The first time that you use the TAMIS POC, you will be asked to provide your name and email address before continuing to the next screen. The purpose of collecting this information is to allow ADOT&PF project sponsors to contact staff who have used the tool to solicit suggestions for improvements and additional functionality. After you have provided this information, you will not be asked to sign in when you use the tool again from the same computer.

Workflow

The TAMIS POC includes a number of pre-defined queries that integrate data from multiple sources to inform asset management and strategic planning decisions. Available panels are described on the right-hand panel of this page. The general workflow is to select, configure, and execute a query. It is then possible to view results in tabular and map form, and to generate summary charts and reports. Specific steps are as follows:

  • Getting started. After you are finished reviewing the information on this introduction page, click on the Let’s Get Started button on this page to begin. This will take you to the Query screen.
  • Selecting and executing a query. On the Query screen, you'll begin by selecting the query you want to execute. After selecting the query, you will be led through a series of questions to filter the query results as desired. Once you have completed the questions, you can execute the query to proceed.
  • Viewing results. After the query has executed, the application will transition to the Results screen, where you’ll be able to view the results in both tabular and map form.
  • Viewing charts. From the Results screen, you can click on the Charts tab if you want to view a series of charts providing visual summaries of the results.
  • Viewing predefined reports. You can also click on the Reports tab if want to view a predefined report that summarizes the results of the query.
  • Creating ad-hoc reports. Finally, you can click on the Explore tab for access to a pivot-table tool that allows you to generate ad-hoc reports based on the data returned from the query.
  • Running additional queries. When you are done examining the results of one query, you can return to the Query tab to select, configure, and execute another query.

Available Queries

  • ADOT&PF Assets. This query allows users to look at road and/or bridge assets for which ADOT&PF has maintenance responsibility. The user can choose to filter results based on region, legislative district, and NHS classification.
  • ADOT&PF Asset Conditions. This query allows the user to examine the current condition ratings of roads and/or bridges for which ADOT&PF has maintenance responsibility. The user can choose to filter results based on region, legislative district, NHS classification, pavement condition, bridge status, and bridge deck condition.
  • Corridor Conditions. This query allows the user to first specify a section of roadway (selecting a given route along with a beginning and ending mile point) and then provides information about the current condition of pavement and bridges that fall along the specified section.
  • Crash Statistics for STIP Projects. This query compiles crash statistics for sections of roadway associated with planned projects in the STIP. The user can filter results based on region or legislative district.
  • Unstable Slopes. This query identifies road sections for which ADOT&PF has maintenance responsibility that may be at risk of closure based on proximity to unstable slopes. The user can choose to filter results based on region, legislative district, traffic volume, minimum total risk-hazard score, and whether or not mitigation measures have been implemented.
  • Creating ad-hoc reports. Finally, you can click on the Explore tab for access to a pivot-table tool that allows you to generate ad-hoc reports based on the data returned from the query.
  • Running additional queries. When you are done examining the results of one query, you can return to the Query tab to select, configure, and execute another query.
